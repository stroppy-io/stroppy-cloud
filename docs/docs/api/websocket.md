# WebSocket API

Stroppy Cloud provides real-time log streaming via WebSocket connections. The WebSocket hub broadcasts three types of messages: DAG executor logs, agent reports, and agent log lines.

## Endpoints

### GET /ws/logs

Connect to receive logs from all runs.

```bash
websocat ws://localhost:8080/ws/logs
```

### GET /ws/logs/{runID}

Connect to receive logs from a specific run only. Messages from other runs are filtered out.

```bash
websocat ws://localhost:8080/ws/logs/run-001
```

## Message Format

All WebSocket messages are JSON with the following envelope:

```json
{
  "type": "<message_type>",
  "run_id": "<run_id>",
  "node_id": "<node_id>",
  "payload": { ... }
}
```

### Message Types

#### type: "log"

DAG executor log entries. Generated by node `Task` implementations via the `NodeContext` logger. These are the primary execution logs.

```json
{
  "type": "log",
  "run_id": "run-001",
  "node_id": "install_db",
  "payload": {
    "level": "info",
    "message": "installing postgresql-16 on db-node-1",
    "time": "2026-03-31T12:00:00Z"
  }
}
```

Log levels follow zap conventions: `debug`, `info`, `warn`, `error`.

#### type: "report"

Agent command completion reports. Broadcast when an agent reports back via `POST /api/agent/report`.

```json
{
  "type": "report",
  "payload": {
    "command_id": "cmd-install-pg-001",
    "status": "completed",
    "output": "postgresql-16 installed"
  }
}
```

The `status` field is one of: `running`, `completed`, `failed`.

#### type: "agent_log"

Raw log lines streamed from agents during command execution. Broadcast when an agent sends a log line via `POST /api/agent/log`.

```json
{
  "type": "agent_log",
  "payload": {
    "command_id": "cmd-install-pg-001",
    "line": "Reading package lists... Done",
    "stream": "stdout"
  }
}
```

## Filtering

When connecting to `/ws/logs/{runID}`, only messages with a matching `run_id` are sent. Messages without a `run_id` (such as agent reports that are not yet associated with a run) are still delivered.

When connecting to `/ws/logs` (no filter), all messages from all runs are delivered.

## Connection Management

- The server uses `gorilla/websocket` with permissive CORS (`CheckOrigin` returns `true`).
- Clients do not need to send any messages; the connection is read-only from the client perspective.
- The server detects client disconnections via the read pump and removes them from the hub.

## Example: JavaScript Client

```javascript
const ws = new WebSocket('ws://localhost:8080/ws/logs/run-001');

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  switch (msg.type) {
    case 'log':
      console.log(`[${msg.node_id}] ${msg.payload.level}: ${msg.payload.message}`);
      break;
    case 'report':
      console.log(`Command ${msg.payload.command_id}: ${msg.payload.status}`);
      break;
    case 'agent_log':
      console.log(`[agent] ${msg.payload.stream}: ${msg.payload.line}`);
      break;
  }
};
```
